# 量化交易系统调研报告与改进计划

## 一、行业量化软件通用架构调研

### 1.1 标准四层架构

根据对行业主流量化平台（聚宽、BigQuant、Qbot、Futures_ML等）的调研，专业量化系统普遍采用以下四层架构：

```
┌─────────────────────────────────────────────────────────────┐
│  研究环境 (Research)                                         │
│  - 特征工程实验                                              │
│  - 模型训练与调参 (MLflow记录)                               │
│  - 单因子IC分析                                              │
│  产物: strategy.yaml + model_artifact + feature_sql         │
├─────────────────────────────────────────────────────────────┤
│  回测环境 (Backtest)                                         │
│  - 遍历逻辑 / 向量化计算                                     │
│  - 滑点、手续费、资金费率                                    │
│  - 仓位管理 (Kelly公式 / 风险平价)                           │
│  - 确定性验证 (同输入同输出)                                 │
│  产物: trades.parquet + metrics.json                         │
├─────────────────────────────────────────────────────────────┤
│  模拟环境 (Paper Trading)                                    │
│  - 与回测引擎同一接口实现                                    │
│  - N天模拟运行偏差报告                                       │
│  产物: deviation_report (与backtest的PnL偏差)                │
├─────────────────────────────────────────────────────────────┤
│  实盘环境 (Live Trading)                                     │
│  - 实时数据接入                                              │
│  - 订单执行与风控                                            │
│  产物: daily_PnL归因 + 异常订单台账 + 滑点对比              │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    实盘数据反馈到研究环境 (闭环)
```

### 1.2 特征工程最佳实践

| 特征类别 | 具体指标 | 用途 |
|---------|---------|------|
| **动量类** | MA5/10/20/60、MACD、RSI、KDJ、ROC | 趋势判断 |
| **波动率类** | ATR、布林带宽度、滚动标准差 | 风险度量 |
| **量价类** | OBV、量价背离、换手率MA、成交量分位 | 资金动能 |
| **统计类** | 偏度、峰度、Z-Score、价格位置 | 极端值识别 |
| **宏观类** | 国债收益率、VIX、汇率 | 市场环境 |
| **滞后类** | 前N日收益率、前N日成交量 | 时序信息编码 |

**关键原则**：特征数 / 样本数 < 1/10，避免过拟合。

### 1.3 模型选择与训练方式

| 模型 | 优势 | 劣势 | 适用场景 |
|------|------|------|---------|
| **LightGBM** | 训练快(6s)、可解释、表格数据强 | 略逊于LSTM的RankIC | 日频策略首选 |
| **XGBoost** | 稳定、泛化强、可SHAP解释 | 训练较慢 | 中频策略 |
| **RandomForest** | 不易过拟合、并行友好 | 精度一般 | 基线模型 |
| **LSTM/GRU** | 时序建模能力强 | 训练慢(34s)、黑盒 | 高频/复杂模式 |
| **Transformer** | 长程依赖捕捉 | 数据量要求大 | 大规模数据 |

**训练方式**：
- **单次建模**：全量数据训练一次，简单但易过拟合
- **推进建模 (Walk-Forward)**：每日收盘后用最新数据重训练，最贴近实盘
- **滚动窗口 (Sliding-Window)**：固定窗口长度滚动训练，平衡稳定性与适应性

### 1.4 回测引擎核心要求

1. **避免未来函数**：信号只能使用当前及之前的数据
2. **确定性**：同代码+同输入+同种子 = 同输出
3. **成本模拟**：滑点 + 手续费（ETF通常万1~万3）
4. **风控模块**：
   - 止损止盈（ATR动态止损）
   - 最大回撤限制
   - 仓位上限（单标的不超过X%）
   - 波动率控制（波动大时降仓）
5. **特殊处理**：节假日、停牌、除权除息

### 1.5 模型评估指标（超越Accuracy）

| 指标 | 说明 | 为什么重要 |
|------|------|-----------|
| **RankIC** | 预测值与实际收益排名的相关系数 | 衡量排序能力，比Accuracy更稳健 |
| **Precision** | 预测涨实际涨的比例 | 减少假阳性，控制回撤 |
| **Recall** | 实际涨被预测出的比例 | 减少踏空 |
| **F1-Score** | Precision与Recall的调和平均 | 平衡指标 |
| **Calmar比率** | 年化收益 / 最大回撤 | 风险调整后收益 |
| **夏普比率** | (收益-无风险利率) / 波动率 | 经典风险调整指标 |
