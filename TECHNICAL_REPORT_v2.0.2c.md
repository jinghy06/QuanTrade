## 三、核心优化：双模式动量 + 防御性ETF排除

### 3.1 双模式动量选股（ETF自身判断）

```python

    is_bottom = mom_120 < -0.15  # ETF自身120日跌幅>15%
    
    if is_bottom:

    else:
        # 正常模式：选长期动量最强的（趋势跟踪）
        score_120 = np.clip(mom_120 / 0.5, -1, 1)

```

### 3.2 防御性ETF排除（关键突破）

**核心发现**：515960（银行ETF）在2024年初市场底部时几乎没跌（120日动量+0.6%），在正常模式下得分最高被选中。但在2024-2026成长股牛市中，银行ETF跌了12%，严重拖累组合收益。

**修复方案**：
1. 传递`symbol`参数到`calculate_momentum_score`（解决`hasattr`检测失败）
2. 在市场底部（`market_bottom`）或自身底部（`is_bottom`）时，排除防御性ETF
3. 选股时只选得分>0的ETF（避免凑数选中低分标的）

```python
DEFENSIVE_ETFS = ['515960', '512010', '512690', '512880', '512800', '512200']
if (is_bottom or market_bottom) and symbol and symbol in DEFENSIVE_ETFS and mom_120 > -0.05:
    return -100  # 极低分，不选中

# 选股时只选得分>0的
positive_scores = [(s, sc) for s, sc in sorted_scores if sc > 0]
top_etfs = positive_scores[:MAX_POSITIONS]
```

### 3.3 为什么515960是收益黑洞？

| ETF | 2024-2026涨幅 | 属性 | 底部模式表现 |
|-----|-------------|------|-------------|
| 516510 | +97% | 车联网/成长 | 超跌后反弹强劲 |
| 515070 | +184% | 红利/科技 | 超跌后反弹强劲 |
| **515960** | **-12%** | **银行/防御** | **抗跌=资金避风港，牛市资金流出** |

**结论**：防御性ETF在市场底部不应被选中，因为：
- 抗跌不是因为强，而是因为资金避险
- 当市场反弹时，资金从防御性ETF流出到成长股
- 防御性ETF在牛市中跑输

## 六、关键代码决策

### 6.1 为什么去掉ML？

- v2.0.1使用LightGBM/RF/GB，但15只ETF仅有2026年数据，无法训练
- 8只完整数据ETF样本太少，ML容易过拟合
- 纯动量+规则策略在样本外表现更稳健（Walk-Forward 80%）

### 6.2 为什么用ETF自身判断底部？

- 市场统一判断（沪深300跌>10%）让所有ETF走底部模式
- 抗跌ETF（如515960）在底部模式下得分反而低，但正常模式下得分高
- ETF自身判断更精准：只有真正超跌的ETF才触发底部模式

### 6.3 为什么等权基准？

- 用户要求基准改为"等权持有和买入不动"
- 等权基准84%收益（2024-2026牛市，8只ETF普涨）
- 策略核心优势：选到最好的2只，避免选中最差的（515960）
